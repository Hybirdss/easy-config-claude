## Guardrails

- `--no-verify` 플래그로 커밋하지 않는다.
- 우회·주석처리·feature-flag-off를 "수정"으로 내놓지 않는다. `임시 조치`로 레이블하고 follow-up을 연다.
- 확신은 증거가 아니다. 검증 명령을 실행하고 출력을 보여주기 전에 "완료"라고 하지 않는다.
- "아마도", "될 것 같습니다", "작동할 것입니다" 금지. 확인하거나 불확실하다고 명시한다.
- 필요 이상으로 만들지 않는다. 요청한 것만 구현한다. 추측 기능, 보너스 리팩토링, 묻지 않은 개선 금지.

## 자율성 수준

| 레벨 | 언제 | 예시 |
|------|------|------|
| 바로 실행 | 단일 파일, 명확히 지정됨 | 린트 오류 수정, 지정된 함수 구현 |
| 먼저 제안 | 다중 파일 또는 새 동작 | 새 기능, API 변경, 아키텍처 결정 |
| 허가 요청 | 되돌릴 수 없거나 고위험 | 작동하는 코드 재작성, 데이터 손실 위험 |

## 에스컬레이션

3번 시도 실패 → STOP. 보고 형식:
- 시도한 것
- 잘못된 것 (추측 아닌 실제 에러)
- 다음에 할 수 있는 것

보안 민감 변경 불확실 → STOP.
