## Model Routing — 자동 판단 규칙

기본값은 **sonnet**. 아래 신호가 감지되면 해당 모델로 전환하고, 해당 작업이 끝나면 sonnet으로 복귀한다.

### Opus로 전환하는 신호

| 신호 | 예시 발화/상황 |
|------|--------------|
| **계획·설계 요청** (opusplan) | "플랜 짜줘", "설계해줘", "아키텍처 어떻게", "DB 스키마", "API 설계" |
| **2회 이상 실패한 디버깅** | 같은 에러가 두 번째 등장, "왜 또 안 되지" |
| **보안 리뷰** | "취약점 찾아줘", "security audit", CSO 관련 |
| **중요 코드 리뷰** | 머지 전 리뷰, "이 코드 괜찮아?" |
| **복잡한 리팩토링 판단** | "어떻게 바꾸는 게 나을까", 트레이드오프 분석 |

**전환 방법:** 해당 작업 직전 `/model opus` 실행. 작업 완료 후 `/model sonnet` 복귀.

### Sonnet 유지 (기본)

일반 구현, 테스트 작성, 파일 수정, PR 설명 작성 등 대부분의 작업.

### Haiku는 서브에이전트 자동 처리

`CLAUDE_CODE_SUBAGENT_MODEL` 미설정 시 서브에이전트는 메인 모델(sonnet) 상속.
탐색·파일 읽기 전용 서브에이전트만 분리하고 싶다면:
```json
"CLAUDE_CODE_SUBAGENT_MODEL": "claude-haiku-4-5-20251001"
```
단, 서브에이전트에 실제 판단을 위임하는 경우 haiku 고정은 품질 저하 위험.
기본은 미설정(sonnet 상속) → 필요할 때만 추가.

### 빠른 전환 레퍼런스

```
/model opus       복잡한 설계·디버깅
/model sonnet     기본 복귀
/cost             현재 세션 비용 확인
```
