## Conventions

- 코드 수정 전에 기존 스타일, 라이브러리, 패턴을 읽는다. 새 패턴보다 기존 패턴을 따른다.
- 파일/헬퍼/의존성을 만들기 전에 먼저 검색한다. 이미 있는 것을 중복 생성하지 않는다.
- 접근 방식 변경 시 구 코드를 삭제한다. 하위 호환 shim·`// removed` 주석 금지.
- 언어 매칭: 사용자가 한국어로 쓰면 한국어로, 영어로 쓰면 영어로.

## Search Before Building

파일, 헬퍼, 유틸리티를 만들기 전에:

```bash
# 비슷한 게 이미 있는지 확인
grep -r "함수명 또는 기능 키워드" src/ --include="*.ts" -l
```

검색 결과가 없을 때만 새로 만든다.
라이브러리 관련이면 공식 문서를 WebSearch로 먼저 확인한다.

## 컨텍스트 관리

| 명령 | 언제 |
|------|------|
| `/clear` | 관련 없는 작업 사이. 오래된 컨텍스트는 모든 메시지에 토큰 낭비. |
| `/compact` | 탐색 완료 후, 마일스톤 완료 후, 주요 컨텍스트 전환 전. |
| `/cost` | 현재 세션 비용 확인. |

컨텍스트가 큰 탐색(파일 5개+)은 서브에이전트에게 위임한다.
서브에이전트는 20개 파일을 읽어도 메인 컨텍스트엔 요약만 반환된다.
